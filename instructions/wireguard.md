# WireGuard VPN Guide for Students

Use this guide if you are outside the clinic network and need cluster access.

## 1) Why you need WireGuard

WireGuard creates a secure VPN tunnel into the Dieterichlab network.
Without this tunnel, SSH and JupyterHub may not be reachable from outside.

## 2) What you need before setup

1. Your cluster username
2. WireGuard app installed on your device
3. Public key generated by the WireGuard app
4. An assigned VPN IP address from Harald Wilhelmi

Contact for VPN setup:

- harald.wilhelmi@uni-heidelberg.de

## 3) GUI setup (Windows and macOS)

1. Install WireGuard from https://www.wireguard.com/install/
2. Open the WireGuard app.
3. Create a new empty tunnel.
4. Save it once so your local key pair is created.
5. Send your public key to Harald and ask for your VPN IP.
6. Update the tunnel configuration to the template below.
7. Save and activate the tunnel.

Use this template (do not change peer public key or endpoint):

```ini
[Interface]
PrivateKey = <do NOT change>
Address = <IP address from Harald>
ListenPort = 51820
MTU = 1280
DNS = 10.250.140.21

[Peer]
PublicKey = hP45m/EB6vd8kuU9ii1VdyZCrJ5XZ1XPPcT5eVfkHUI=
Endpoint = 129.206.148.250:51820
AllowedIPs = 129.206.148.251/32,10.250.135.0/24,10.250.140.0/24,10.250.100.0/24
```

## 4) First access test

After activating WireGuard, test cluster access:

```bash
ssh <username>@cluster.dieterichlab.org
```

If this works, your VPN setup is ready.

## 5) Quick troubleshooting

### Tunnel is active, but SSH fails

- Re-check your assigned IP address in the tunnel config.
- Confirm the `AllowedIPs` entries match the template.
- Deactivate and reactivate the tunnel.

### DNS/internal host resolution fails

- Ensure `DNS = 10.250.140.21` is present in `[Interface]`.
- Restart the WireGuard tunnel.

### Public/private key confusion

- You send only the public key to Harald.
- Never share your private key.

### Still not working

Send the following to instructors/admin:

1. Screenshot of tunnel status (keys can remain hidden)
2. Exact SSH command used
3. Exact error message
